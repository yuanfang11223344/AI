## 生成规则

1. 每日 10:00 由 Codex automation `ai-daily-news` 触发。
2. 每次触发都必须生成当天日报 `YYYY-MM-DD.md`。
3. 周一 10:00 且本周 `weekly-YYYY-WXX.md` 不存在时，在生成日报后额外生成周报。
4. 如果周一周报已存在，只生成或刷新当天日报，不重复生成周报。
5. 日报文件名为 `YYYY-MM-DD.md`，周报文件名为 `weekly-YYYY-WXX.md`。
6. 每条资讯必须有来源链接；论文必须有 arXiv 链接；芯片与互连板块必须独立。
7. 全文中文，专业术语保留英文，不编造论文结论。
8. 大模型、LLM、推理框架、训练方法、模型架构、Agent 模型能力更新必须进入“大模型与 LLM 技术解读”板块。

## 运行前预检补档规则

1. 每次手动或自动更新 AI 资讯时，必须先检查近期日报和周报是否缺失，再生成当天应生成的报告。
2. 近期检查窗口默认覆盖当天往前 10 个自然日；如这 10 天内包含周一，还必须检查对应 `weekly-YYYY-WXX.md` 是否存在。
3. 缺失日报按日期从旧到新补齐；缺失周报在其对应周一日报已存在后补齐。
4. 补齐历史缺口后，才生成或刷新当天日报；如果当天是周一，还要继续执行“当天日报 + 本周周报”的规则。
5. 执行报告必须写明检查日期范围、发现的缺失文件、实际补齐文件、跳过的已存在文件和 Git 上传结果。
6. 如果用户在对话中新增或修改 AI 资讯更新规则，必须在同一次维护中及时写入本文件、`settings.yaml` 或自动化 prompt 中的相应位置，并同步到 Obsidian 与 AI.git 维护文件。

## Git 规则

- `AI资讯仓库` 对应远端：`https://github.com/yuanfang11223344/AI.git`。
- AI 资讯自动化按用户授权，生成报告后可以执行 `git add`、`git commit` 和 `git push` 上传到 AI.git。
- 自动提交范围仅限 AI 资讯报告与 AI.git 维护文件，不提交缓存、下载残留或无关目录。
- 不提交 `.DS_Store`、临时下载文件、缓存文件或与 AI 资讯无关的内容。
- Git push 失败时必须在执行报告中写明错误，不静默跳过。

## LLM 技术解读模板

日报和周报遇到大模型或 LLM 技术进展时，必须用以下结构写一段技术总结：

- 背景：这项进展发生在什么技术/产业上下文中。
- 问题：它试图解决什么瓶颈，例如推理成本、长上下文、工具调用、多模态、训练稳定性、数据效率或评测可信度。
- 方法：模型、算法、系统或工程方案是什么，尽量写出关键机制，不只写营销名称。
- 效果：公开材料声称取得了什么效果，必须标注依据；如果只有发布声明而无可验证数据，要明确说“暂无独立评测”。

不要把模型发布只写成“某公司发布某模型”。如果信息不足，仍要说明缺失了哪些关键细节。
