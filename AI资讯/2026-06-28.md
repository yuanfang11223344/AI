## 🧠 大模型与 LLM 技术解读

### 1. 🔥 GPT-5.6 分层模型与受限发布
- **背景**：frontier model 同时面对能力升级、推理成本和国家级网络安全审查。
- **问题**：单一旗舰模型难以兼顾复杂推理、低成本批处理和高吞吐；高能力模型又可能扩大 cyber misuse 风险。
- **方法**：公开报道将系列分为 Sol、Terra、Luna，并先向获批客户进行有限预览。
- **效果**：二手报道提到 Sol 在 Terminal-Bench 达到较高成绩，但官方完整技术报告和统一独立评测尚未公开，不能据此断言总体领先。[AP](https://apnews.com/article/065d5398baac7f16c8265c2cb8ba2baa)

### 2. 🔥 Fable 5 的能力发布与安全路由
- **背景**：Claude 系列面向 coding、scientific research 和长程 agent 任务继续提升能力。
- **问题**：更强 cyber 能力与广泛商用之间存在访问控制冲突。
- **方法**：Anthropic 采用 safeguard-triggered routing，将部分高风险请求转交较低能力模型，并配合分阶段访问。
- **效果**：厂商称触发比例平均低于 5%，但能力和安全效果主要来自官方材料，暂无完整独立评测。[Anthropic](https://www.anthropic.com/news/claude-fable-5-mythos-5)

### 3. ⭐ Jalapeño 的 model-system-silicon co-design
- **背景**：LLM inference 的成本越来越受 memory bandwidth、interconnect 和 serving utilization 约束。
- **问题**：通用 accelerator 难以针对固定模型路线和产品 SLA 做极致优化。
- **方法**：OpenAI 定义模型、kernel 与 serving 需求，Broadcom/Celestica 负责芯片实现及 board/rack/network 集成。
- **效果**：方向上可减少软硬件错配，但制程、HBM、SerDes、tokens/s 和能效数据未公开，暂无独立评测。[OpenAI](https://openai.com/index/openai-broadcom-jalapeno-inference-chip/)

### 4. ⭐ MCP stateless core
- **背景**：Agent 从本地 demo 走向跨服务、长任务和多租户部署。
- **问题**：强依赖 session 的 MCP server 难以水平扩展，也增加恢复与权限治理复杂度。
- **方法**：2026-07-28 RC 将 stateless transport core 与 Tasks、Apps 等扩展分层。
- **效果**：有利于云端扩缩容和协议演进，但旧 server 迁移成本与互操作表现仍需真实部署验证。[MCP RC](https://github.com/modelcontextprotocol/modelcontextprotocol/releases/tag/2026-07-28-RC)
