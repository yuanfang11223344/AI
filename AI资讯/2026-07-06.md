## 🧠 大模型与 LLM 技术解读
### 1. Sonnet 5 的 task economics
- **背景**：长程 Agent 的 token 消耗使单次调用价格失去代表性。
- **问题**：更便宜的 token 若带来更多重试或更长轨迹，任务总成本反而可能上升。
- **方法**：Sonnet 5 以 Sonnet 档位提供 browser、planning、coding 和 knowledge work 能力。
- **效果**：官方称接近 Opus 4.8；早期社区反馈不一致，应以任务成功率、总 tokens、latency 做内部评测。[Axios](https://www.axios.com/2026/06/30/anthropic-sonnet-5-agents-mythos-fable)

### 2. inference-scaled evaluation
- **背景**：ICML 周期内，模型评测的 compute budget 成为关键变量。
- **问题**：固定预算会把资源不足误判成能力不足。
- **方法**：研究改变 token budget、context compaction 和重复提交。
- **效果**：模型排名对 protocol 敏感，采购评测必须披露 inference compute。[arXiv](https://arxiv.org/abs/2606.17930)

### 3. RiVER
- **背景**：RLVR 依赖可判定答案，难覆盖真实工程优化。
- **问题**：连续 score 任务没有唯一 ground truth。
- **方法**：将 deterministic feedback 转为候选排序并进行 group-relative RL。
- **效果**：作者报告可改善 score-based tasks；抗 reward hacking 仍待复现。[arXiv](https://arxiv.org/abs/2606.27369)

### 4. MCP stateless runtime
- **背景**：Agent 工具链进入多租户与长任务部署。
- **问题**：连接状态耦合阻碍弹性扩缩容和故障恢复。
- **方法**：协议 RC 分离 stateless core、Tasks 和 Apps。
- **效果**：生产架构更清晰，但旧实现迁移与授权兼容仍需验证。[MCP](https://github.com/modelcontextprotocol/modelcontextprotocol/releases/tag/2026-07-28-RC)
