## 🧠 大模型与 LLM 技术解读

- 🔥 **Claude Fable 5 / Mythos 5 把“长任务自主执行”推到更前台**
  - 背景：frontier labs 正在把模型能力从单轮问答转向长时程 software engineering、knowledge work、vision 与 cyber workflows。
  - 问题：模型在长任务中常见的瓶颈是 token 效率、记忆保持、持续规划以及高风险领域的安全访问控制。
  - 方法：Anthropic 将 Fable 5 作为面向通用用户的 Mythos-class 模型发布，并对敏感方向加保守 safeguards；对受信任 cyberdefenders 则提供限制更少的 Mythos 5，通过 trusted-access 方案部署。
  - 效果：Anthropic 声称 Fable 5 在其公开比较中对 software engineering、knowledge work、vision 和长任务表现更强，且在不到 5% 的 session 中触发 safeguard；但当前访问已因政府 directive 暂停，暂无独立、持续的公开评测。来源：[Anthropic 发布](https://www.anthropic.com/news/claude-fable-5-mythos-5)、[Anthropic 暂停说明](https://www.anthropic.com/news/fable-mythos-access)

- 🔥 **OpenAI Daybreak 把 LLM 安全能力推进到 patch automation**
  - 背景：随着模型在漏洞发现上的能力上升，安全团队的瓶颈从“发现漏洞”转向“验证并修复漏洞”。
  - 问题：如果模型只能生成 alerts，而不能完成 validation、patch generation、triage 和 workflow integration，企业很难把它纳入真实安全生产流程。
  - 方法：OpenAI 在 Daybreak 中更新 Codex Security plugin，并向 trusted defenders 扩展 GPT-5.5-Cyber，配合 Patch the Planet 把扫描、证据收集、补丁生成和导出到安全系统连接起来。
  - 效果：OpenAI 披露 Codex Security 自 3 月以来已扫描 3000 万+ commits、覆盖 3 万+ codebases，并有 50 万+ findings 被自动判定为已修复；这些数据来自官方产品运行统计，暂无第三方独立审计。来源：[OpenAI Daybreak](https://openai.com/index/daybreak-securing-the-world/)

- ⭐ **DiffusionGemma 试图用 text diffusion 改写生成速度约束**
  - 背景：autoregressive LLM 逐 token 解码的串行性，已经成为 latency-sensitive inference 的长期瓶颈。
  - 问题：在保持可用质量的同时，如何降低生成延迟、提高 GPU 利用率，是部署端最现实的系统问题之一。
  - 方法：Google 推出 experimental open model DiffusionGemma，采用 text diffusion 而非典型 token-by-token decoding，并以 26B MoE、Apache 2.0 license 方式开放。
  - 效果：Google 声称该模型可在 GPU 上实现最高 4x 更快的 text generation；但该路线仍处于实验阶段，生产稳定性、质量折中和生态适配仍需更多独立验证。来源：[Google Blog](https://blog.google/innovation-and-ai/technology/developers-tools/diffusion-gemma-faster-text-generation/)

- ⭐ **Gemini 3.5 Live Translate 把多语言 speech-to-speech 推向实时交互**
  - 背景：多模态模型竞争正从 text + image 延伸到音频实时交互，尤其是 voice agents、跨语种会议与 customer support。
  - 问题：传统语音翻译常在时延、语气保真和多语言泛化之间做妥协，难以支撑自然对话。
  - 方法：Google 将 Gemini 3.5 Live Translate 定位为 audio model，提供 70+ 语言的 near real-time speech-to-speech translation。
  - 效果：Google 的公开表述强调更自然的 voice translation 与大规模语言覆盖，但尚未给出统一公开 benchmark；因此目前更适合作为产品能力跟踪，而不是已被独立验证的通用 SOTA 结论。来源：[Google Blog](https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-live-3-5-translate/)

- ⭐ **Gemma 4 12B 用 encoder-free multimodal 争夺本地 agent 位置**
  - 背景：边缘侧与本地运行模型的竞争点，正在从“能不能跑”转向“能否在有限显存内跑多模态 agent workflows”。
  - 问题：传统 multimodal 模型依赖独立 vision/audio encoders，会增加延迟和内存开销，不利于 laptop-ready 部署。
  - 方法：Google 推出 Gemma 4 12B，采用 unified、encoder-free multimodal architecture，把 audio 和 vision 直接并入 LLM backbone，并加入 Multi-Token Prediction drafters 降低延迟。
  - 效果：Google 声称它接近 26B MoE 的 benchmark 表现，同时可在 16GB 内存/显存级别设备上运行，并以 Apache 2.0 发布；但公开材料仍以官方 benchmark 为主，暂无充分独立评测。来源：[Google Blog](https://blog.google/innovation-and-ai/technology/developers-tools/introducing-gemma-4-12b/)
