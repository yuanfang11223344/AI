## 🧠 大模型与 LLM 技术解读

### 1. 🔥 Claude Sonnet 5：用模型分层降低 Agent 单位任务成本

- **背景**：长程 coding 和工具调用任务需要大量 input/output tokens，旗舰模型虽强，但难以覆盖高频生产流量。
- **问题**：企业需要在 reasoning、上下文、延迟与价格之间取得更可持续的平衡。
- **方法**：公开汇总称 Sonnet 5 提供 1M context、128K max output，并以低于 Opus 的分层定价承接 agentic workloads；关键参数仍应以 Anthropic 正式 model card/API 文档为准。
- **效果**：媒体称其接近 Opus 4.8 的部分 agent 能力，但目前主要是发布材料与早期观察，暂无完整独立横评。来源：[7 月 1 日汇总](https://kenove.no/digest/2026-07-01)

### 2. 🔥 Fable 5 / Mythos 5：模型能力之外，供给治理成为系统变量

- **背景**：高能力模型在 cybersecurity 和自主执行上的风险引发政府审查。
- **问题**：模型即使技术上可用，也可能因审批、地域或安全政策突然中断。
- **方法**：政府解除限制后分阶段恢复访问，厂商继续结合访问控制与 safeguard routing 管理高风险请求。
- **效果**：用户重新获得模型访问，但此次事件证明企业不能只依赖单一 provider；安全审核的长期透明度与可重复性仍待观察。来源：[TechCrunch](https://techcrunch.com/2026/06/30/trump-drops-restrictions-on-anthropics-mythos-and-fable-models/)｜[Anthropic](https://www.anthropic.com/news/claude-fable-5-mythos-5)

### 3. ⭐ Claude Science：从通用聊天走向科研工作流

- **背景**：AI for Science 正从文献问答扩展到 hypothesis generation、数据分析和可追溯研究协作。
- **问题**：科研任务需要引用可核查、长上下文组织、工具执行和实验过程留痕，普通聊天界面难以形成闭环。
- **方法**：Claude Science 将模型能力与专用科研工作流结合，强化资料整合、分析与研究任务协作。
- **效果**：产品定位清晰，但跨学科准确性、科研产出提升和失败模式尚缺少独立、长期评估。来源：[AI Round-up](https://www.ai-roundup.dev/)

### 4. ⭐ MCP stateless core：为生产 Agent 降低会话耦合

- **背景**：MCP server 正从本地工具连接器进入多租户、弹性部署和长任务场景。
- **问题**：强 session 状态增加水平扩展、故障恢复和权限治理难度。
- **方法**：2026-07-28 RC 将 stateless transport core 与 Tasks、Apps 等扩展分层，减少核心协议对连接状态的依赖。
- **效果**：架构上更利于云端扩缩容，但旧 server 的迁移成本、授权兼容和跨实现互操作仍需验证。来源：[MCP GitHub](https://github.com/modelcontextprotocol/modelcontextprotocol/releases/tag/2026-07-28-RC)
