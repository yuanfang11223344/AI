## 🧠 大模型与 LLM 技术解读

1. 🔥 GPT-5.6 family：多档模型服务同一产品线
   - 背景：企业和开发者同时需要最高能力、稳定日常模型和低成本模型。
   - 问题：单一 frontier model 难以兼顾 coding/agent 复杂任务、办公交互和高并发低成本场景。
   - 方法：OpenAI 将 GPT-5.6 拆成 Sol、Terra、Luna，并在 ChatGPT / API / Microsoft 365 Copilot 等入口分发。
   - 效果：官方声称 Sol 是 flagship，Luna 是成本最低版本；公开材料为发布声明，暂无独立评测。来源：[OpenAI](https://openai.com/index/gpt-5-6/)、[OpenAI Copilot](https://openai.com/index/gpt-5-6-preferred-model-microsoft-365-copilot/)
2. 🔥 ChatGPT Work：agent 从 coding IDE 扩展到办公执行层
   - 背景：Codex/Claude Code 已验证 coding agent 需求，下一步是面向普通知识工作流。
   - 问题：跨 Slack、Gmail、Drive、CRM 等系统执行任务时，权限、引用、回滚和审计都会成为风险。
   - 方法：媒体报道 ChatGPT Work 将 GPT-5.6 模型族接入桌面端和 connected apps，生成文档、表格、演示和应用原型。
   - 效果：当前缺少公开任务成功率和安全评测；需要跟踪企业版权限模型。来源：[The Verge](https://www.theverge.com/ai-artificial-intelligence/963464/openai-gpt-5-6-codex-chatgpt-work)
3. ⭐ Claude Sonnet 5：中档价格承接更强 agent 能力
   - 背景：Sonnet 系列长期是 Claude Code 与工具调用的主力模型。
   - 问题：Opus 档能力强但成本高，企业需要更低成本的持续 agent worker。
   - 方法：Anthropic 称 Sonnet 5 可计划、使用浏览器和终端工具，并支持更高 effort level。
   - 效果：官方称部分场景接近 Opus 4.8，价格低于 Opus；暂无独立长期 agent 可靠性评测。来源：[Anthropic](https://www.anthropic.com/news/claude-sonnet-5)
4. ⭐ GPT-Live：全双工语音模型进入 ChatGPT Voice
   - 背景：语音 AI 的瓶颈不只是 ASR/TTS，而是低延迟双向互动。
   - 问题：传统 turn-taking 语音助手容易打断不自然，长对话状态也难维护。
   - 方法：OpenAI 称 GPT-Live 使用 full-duplex architecture，可同时听和说。
   - 效果：官方发布说明强调自然交互；真实延迟、噪声鲁棒性和多语言表现仍需独立体验验证。来源：[OpenAI](https://openai.com/index/introducing-gpt-live/)
5. ⭐ Gemini Omni Flash：统一多模态生成与创作入口
   - 背景：视频、图像、音频和文本创作模型正在合并到统一产品体验。
   - 问题：多模态模型需要在低延迟、创作质量和 API 可用性之间折中。
   - 方法：Google 将 Gemini Omni Flash 推向订阅用户，并向开发者/企业开放 public preview。
   - 效果：公开材料为产品可用性信息，暂无独立 benchmark；可关注 API 稳定性和区域限制。来源：[Google DeepMind](https://deepmind.google/blog/introducing-gemini-omni/)、[Google DeepMind](https://deepmind.google/blog/start-building-with-nano-banana-2-lite-and-gemini-omni-flash/)
