## 🧠 大模型与 LLM 技术解读

1. 🔥 GPT-5.6 Sol/Terra/Luna：能力分层与成本分层同时产品化
   - 背景：OpenAI 在 limited preview 之后公开 GPT-5.6 family，并把模型分为 Sol、Terra、Luna 三档。
   - 问题：单一旗舰模型难以覆盖高端 reasoning、日常办公、低成本高频调用和 agentic coding 的不同预算。
   - 方法：OpenAI 将 Sol 定位旗舰，Terra 面向 everyday work，Luna 主打 cost efficiency；Axios 还报道 Sol ultra mode 可更努力推理并委派 submodels。
   - 效果：Barron’s 报道 Sol 在第三方榜单中位于高位且成本更低；Axios 引述 54% coding token efficiency。来源为官方和媒体/benchmark，仍需更多独立实测。来源：[OpenAI](https://openai.com/index/gpt-5-6/)、[Axios](https://www.axios.com/2026/07/09/ai-openai-gpt-release)、[Barron’s](https://www.barrons.com/articles/openai-gpt-5-6-anthropic-google-spacex-7a511e79)
2. 🔥 ChatGPT Work：从 chatbot 到跨应用办公 agent
   - 背景：企业 AI adoption 的瓶颈是把模型接入真实文件、应用、权限和可审计流程。
   - 问题：模型如果只能聊天，就难以完成文档、表格、演示、检索与跨系统执行的闭环。
   - 方法：Axios 报道 ChatGPT Work 由 GPT-5.6 驱动，可跨 connected apps/files 收集上下文，生成 documents、spreadsheets、presentations，并先进入 Mac/Windows apps。
   - 效果：公开信息是产品发布，暂无独立生产场景可靠性评测；关键风险在权限、数据边界和输出审计。来源：[Axios](https://www.axios.com/2026/07/09/ai-openai-gpt-release)
3. ⭐ Deep Research source attribution：citation verifier 不一定需要 frontier model
   - 背景：RAG 和 deep research 产品需要模型给出可追溯引用，错误引用会直接破坏可信度。
   - 问题：frontier model 做 verifier 成本高，且不一定比专门 rubric/评测模型稳定。
   - 方法：arXiv 论文 `Do You Need a Frontier Model as a Citation Verifier?` 关注 deep-research source attribution 的 rubric LLM benchmarking。
   - 效果：当前依据为 arXiv 条目，具体结论需阅读全文；方向上说明引用评测将成为企业知识 agent 的核心指标。来源：[arXiv:2607.08700](https://arxiv.org/abs/2607.08700)、[arXiv cs.CL recent](https://arxiv.org/list/cs.CL/recent)
4. ⭐ Proactive agents benchmark：从被动响应走向主动任务执行
   - 背景：ChatGPT Work、Claude Code、Codex 等都在推动 agent 主动规划和跨工具执行。
   - 问题：传统 QA benchmark 不足以衡量 proactive agent 是否能识别时机、调用工具、推进真实任务。
   - 方法：`UniClawBench` 是面向 real-world proactive agents 的 universal benchmark，arXiv 条目附带项目页与 GitHub。
   - 效果：公开条目表明社区正在为 proactive agents 建立标准化评测；实际覆盖任务和评分质量需看论文与代码。来源：[arXiv:2607.08768](https://arxiv.org/abs/2607.08768)
5. ⭐ Speculative decoding 新变体：DominoTree 针对 conditional tree-structured drafting
   - 背景：高端 LLM 走向 agent 后，延迟与吞吐成为用户体验和成本核心。
   - 问题：单一路径 speculative decoding 对复杂生成和分支推理支持有限。
   - 方法：`DominoTree` 研究 conditional tree-structured drafting with Domino for speculative decoding，尝试通过树状草稿提升解码效率。
   - 效果：arXiv 条目显示论文含代码链接；是否能稳定提升实际 serving 吞吐要看实验和工程实现。来源：[arXiv:2607.08642](https://arxiv.org/abs/2607.08642)
6. 📌 Prompt Compression via Activation Aggregation：长上下文成本继续被压缩
   - 背景：1M token context 正成为高端模型标配，但长上下文推理成本和 latency 仍高。
   - 问题：如何压缩 prompt，同时保留任务所需信息。
   - 方法：论文标题显示方法是 activation aggregation 做 prompt compression。
   - 效果：当前仅据 arXiv 条目，具体压缩率和下游损失需阅读全文验证。来源：[arXiv:2607.08399](https://arxiv.org/abs/2607.08399)
