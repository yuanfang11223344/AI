## 🧠 大模型与 LLM 技术解读

1. 🔥 Distillation/IP 争议：模型输出成为新训练数据边界
   - 背景：前沿模型输出可被低成本模型用作蒸馏数据，开源和商业模型能力差距缩小。
   - 问题：如何区分合法 distillation、用户学习和大规模未授权复制；同时 AI 公司自身又依赖互联网数据。
   - 方法：媒体报道美国公司通过账号行为、输出模式和访问日志追踪可疑抽取，并推动监管。
   - 效果：目前主要是公司指控与媒体报道，暂无公开技术审计；但会影响 API rate limit、输出水印和合规条款。来源：[Business Insider](https://www.businessinsider.com/ai-giants-learn-hard-truth-modern-internet-anthropic-openai-google-2026-7)、[New York Post](https://nypost.com/2026/07/13/business/how-china-is-ripping-off-cutting-edge-ai-from-anthropic-openai-and-threatening-us-national-security/)
2. 🔥 GPT-5.6 production migration：成本指标进入模型选择核心
   - 背景：模型发布后，企业不只看 benchmark，而看真实 agent workflow 的 wall-clock、token 和失败重试。
   - 问题：frontier model 更强但更贵，是否能降低总任务成本取决于执行步数、上下文长度和工具调用可靠性。
   - 方法：HN 热帖报告一个生产 agent 迁移到 GPT-5.6 后速度和成本改善，社区重点讨论可复现性与 workload 依赖。
   - 效果：该案例是单方经验，不等同独立基准；但说明 task economics 已替代 token 单价成为迁移依据。来源：[Hacker News](https://news.ycombinator.com/item?id=48882716)
3. ⭐ GPT-5.6 in Microsoft 365 Copilot：frontier model 嵌入办公套件
   - 背景：生产力套件是企业 AI 最大分发渠道之一。
   - 问题：办公场景需要长文档理解、表格推理、代码/公式生成和权限边界。
   - 方法：OpenAI 称 GPT-5.6 为 Microsoft 365 Copilot preferred model，覆盖 Word、Excel、PowerPoint、Chat、Cowork。
   - 效果：官方称能力更强、工作质量更高；暂无独立企业级准确率/安全评估。来源：[OpenAI](https://openai.com/index/gpt-5-6-preferred-model-microsoft-365-copilot/)
4. ⭐ Future Confidence Distillation：把置信度建模提前到生成过程
   - 背景：RAG、tool use、adaptive compute 需要模型知道自己何时不确定。
   - 问题：多数 confidence 方法只看最终答案，忽略生成中置信信息如何变化。
   - 方法：论文提出 Future Confidence Distillation，建模回答过程中的未来置信信号。
   - 效果：具体实验需以论文为准；方向上有助于检索触发、工具调用和拒答策略。来源：[arXiv:2607.07626](https://arxiv.org/abs/2607.07626)
5. ⭐ Albireo inference system：推理效率继续从 serving 系统突破
   - 背景：长上下文和 agent workload 让 GPU 调度与 I/O 成为瓶颈。
   - 问题：Amdahl 限制下，非并行部分会吞噬硬件扩展收益。
   - 方法：Albireo 通过调度/I/O 与 compute overlap、sequence-parallel sampling 等方式提升 parallel inference。
   - 效果：论文摘要声称相对 vLLM 有吞吐、延迟、GPU 利用率和能耗改进；需独立复现。来源：[arXiv:2606.01927](https://arxiv.org/abs/2606.01927)
