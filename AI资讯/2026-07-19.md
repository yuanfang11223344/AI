## 🧠 大模型与 LLM 技术解读

1. 🔥 GPT-5.6 Sol 的 agentic reasoning 与 subagent 编排
   - 背景：OpenAI 在 GPT-5.6 页面中把 Sol 定位为面向复杂任务、企业自动化和 agent development 的 frontier model。
   - 问题：长链工程任务的瓶颈不只是单步推理，而是任务分解、工具调用、子代理协同和中间状态保持。
   - 方法：公开材料强调 reasoning、decision making、autonomy 和 subagent use 的提升；从用户反馈看，重点是把复杂工作拆给多个 agent 子流程执行。
   - 效果：OpenAI 给出合作伙伴评价和产品集成信号，但没有公开完整独立评测；对工程团队而言，仍需用真实仓库任务和成本曲线复测。[来源链接](https://openai.com/index/gpt-5-6/)

2. 🔥 GPT-5.6 Sol Pro 数学证明案例暴露“强模型 + 强提示工程”的新范式
   - 背景：Reddit/r/math 与 HN 讨论称，GPT-5.6 Sol Pro 在一次约 148 分钟会话中给出凸优化开放问题证明，并用 Lean 形式化验证。
   - 问题：数学与 EDA 类研究最难的环节是把非形式化直觉、检索、证明草稿和形式验证连成闭环。
   - 方法：案例不是简单零样本输出，而是使用长 prompt、问题背景压缩、可能的 AI 辅助 prompt 设计和 Lean verification。
   - 效果：社区认为结果有信号价值，但强依赖领域专家 scaffold，不能等同于模型独立完成研究；暂无广泛独立复现。[来源链接](https://www.reddit.com/r/math/comments/1uxj3cy/after_openais_cdc_proof_announcement_gpt56_used_a/) / [HN 讨论](https://news.ycombinator.com/item?id=48939768)

3. 🔥 Claude Fable 5/ Mythos 5 的安全分层与再部署
   - 背景：Anthropic 因美国出口管制一度暂停 Claude Fable 5 与 Mythos 5，随后在限制解除后重新开放 Fable 5。
   - 问题：frontier model 的 cyber capability 使“全球可用”和“高风险能力控制”之间出现冲突。
   - 方法：Anthropic 将 Fable 5 作为带强 safeguards 的通用模型重新部署，同时把 Mythos 5 留在受限客户与 Project Glasswing 场景。
   - 效果：Fable 5 恢复全球可用，但公开材料没有提供完整第三方安全评测；关键细节仍是 safeguard 覆盖率、误拒率和绕过成本。[来源链接](https://www.anthropic.com/news/redeploying-fable-5) / [模型文档](https://platform.claude.com/docs/en/about-claude/models/introducing-claude-fable-5-and-claude-mythos-5)

4. ⭐ Claude Sonnet 5 的“effort level + 价格”产品化
   - 背景：Anthropic 发布 Sonnet 5，并把价格、rate limits 和不同 effort levels 放在公告重点位置。
   - 问题：企业和个人开发者越来越关心单位 token 成本、延迟、上下文长度与结果稳定性，而不是只看最强模型。
   - 方法：Sonnet 5 通过可选 effort level 和阶段性优惠价格，把推理深度与成本控制显式产品化。
   - 效果：公告给出了价格与可用性，但暂无跨模型、跨任务的独立性价比评测；适合在代码生成、长文档分析和 agent 工具调用上做本地基准。[来源链接](https://www.anthropic.com/news/claude-sonnet-5)

5. ⭐ Meta Muse Spark 1.1 的多模态 agentic coding 路线
   - 背景：Meta Muse Spark 系列从个人 superintelligence 叙事切入，强调多模态推理、tool-use、visual chain of thought 与 multi-agent orchestration。
   - 问题：Meta 需要在 OpenAI/Anthropic 主导的 coding agent 市场证明自身模型可用于真实开发流程，而不仅是社交应用助手。
   - 方法：Muse Spark 1.1 被定位到 agentic coding，并通过 Meta AI 应用、社交端入口和 API private preview 扩散。
   - 效果：第三方报道称其面向 coding/agent benchmark 竞争；但公开可复现实验、开放权重和企业集成细节仍不足，暂无独立评测。[来源链接](https://ai.meta.com/blog/introducing-muse-spark-msl/) / [TechCrunch](https://techcrunch.com/2026/07/09/meta-enters-the-crowded-ai-coding-battle-with-muse-spark-1-1/)
