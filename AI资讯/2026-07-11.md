## 🧠 大模型与 LLM 技术解读

1. 🔥 GPT-5.6 family：frontier model 从单旗舰走向多档产品矩阵
   - 背景：前沿模型竞争进入高频发布阶段，企业同时要求最高能力、稳定日常使用和可控成本。
   - 问题：单一模型无法同时满足复杂 coding/agent、日常办公和低成本高并发调用。
   - 方法：OpenAI 官方将 GPT-5.6 分为 Sol、Terra、Luna；Axios 报道 Sol ultra mode 可投入更多计算并委派 submodels，ChatGPT Work 则把模型嵌入办公应用。
   - 效果：媒体称 Sol 在 benchmark 与 cost per task 上表现强；54% token efficiency 是公司公开说法，独立评测仍需继续跟踪。来源：[OpenAI](https://openai.com/index/gpt-5-6/)、[Axios](https://www.axios.com/2026/07/09/ai-openai-gpt-release)、[Barron’s](https://www.barrons.com/articles/openai-gpt-5-6-anthropic-google-spacex-7a511e79)
2. 🔥 ChatGPT Work：agent 产品进入跨应用执行层
   - 背景：企业希望 AI 不只是回答问题，而是能读取上下文、生成文档、推进流程。
   - 问题：跨 app/files 的 agent 容易遇到权限、数据泄露、不可解释操作和任务失败恢复问题。
   - 方法：Axios 报道 ChatGPT Work 可跨 connected apps/files 收集上下文并生成 documents、spreadsheets、presentations，先进入 Mac/Windows apps。
   - 效果：公开材料尚无独立生产可靠性数据；可验证指标应包括任务完成率、权限隔离、引用准确率和回滚能力。来源：[Axios](https://www.axios.com/2026/07/09/ai-openai-gpt-release)
3. ⭐ NemoClaw / enterprise agent mesh：agent 治理成为基础设施问题
   - 背景：OpenClaw 等开源 autonomous agent 激发开发者采用，但企业担心不可控权限和安全责任。
   - 问题：如果 agent 能跨系统执行，企业需要策略执行、审计、边界、供应商互操作，而不是只要更强模型。
   - 方法：TechRadar 称 Nvidia NemoClaw 在 OpenClaw 基础上加入 enterprise-grade reliability、policy enforcement、infrastructure governance，并不强制依赖 Nvidia silicon。
   - 效果：这是媒体解读与产品定位，暂无独立大规模部署数据；但方向清楚：agent 的 moat 会来自治理和平台集成。来源：[TechRadar](https://www.techradar.com/pro/why-nvidias-nemoclaw-signals-the-true-enterprise-agent-era)
4. ⭐ UniClawBench：proactive agents 需要真实任务评测
   - 背景：agent 从“被动聊天”走向“主动规划、工具调用、持续执行”。
   - 问题：传统 benchmark 难以衡量主动发现任务、选择工具、跨步骤保持状态和正确结束的能力。
   - 方法：`UniClawBench` 定位 universal benchmark for proactive agents on real-world tasks，arXiv 条目提供项目页与 GitHub。
   - 效果：当前为新论文/benchmark，实际可信度取决于任务覆盖、评分器稳定性和可复现代码。来源：[arXiv:2607.08768](https://arxiv.org/abs/2607.08768)
5. ⭐ LLM-as-Judge 与 source attribution：评测层开始成为系统瓶颈
   - 背景：Deep Research、RAG、agent 报告都依赖引用与 judge model。
   - 问题：评测器变化会改变测量结果，引用 verifier 如果成本过高也难以规模化。
   - 方法：`When the Judge Changes...` 审计 LLM-as-Judge reliability；`Do You Need a Frontier Model as a Citation Verifier?` 比较 rubric LLMs 做 source attribution。
   - 效果：这些工作说明评测可信度和引用准确性正在成为产品级指标；具体结论需阅读全文，不应只凭标题断言。来源：[arXiv:2607.08535](https://arxiv.org/abs/2607.08535)、[arXiv:2607.08700](https://arxiv.org/abs/2607.08700)
6. 📌 Inference efficiency：speculative decoding、prompt compression 与 pruning 同时升温
   - 背景：1M context、agentic coding 和 enterprise workloads 让推理成本迅速上升。
   - 问题：需要在不明显牺牲质量的情况下降低 latency、KV/prompt 成本和模型计算。
   - 方法：近期 arXiv 包括 `DominoTree` speculative decoding、`Prompt Compression via Activation Aggregation`、`Structured Pruning...` 等。
   - 效果：均需以论文实验为准；趋势上说明模型竞争已从参数规模扩展到 serving efficiency。来源：[arXiv:2607.08642](https://arxiv.org/abs/2607.08642)、[arXiv:2607.08399](https://arxiv.org/abs/2607.08399)、[arXiv:2607.08027](https://arxiv.org/abs/2607.08027)
