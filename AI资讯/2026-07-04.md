## 🧠 大模型与 LLM 技术解读
### 1. Claude Sonnet 5
- **背景**：Agent 任务消耗大量 tokens，企业需要比 Opus 更经济的默认执行模型。
- **问题**：峰值能力、延迟、价格和 cyber 风险难以同时优化。
- **方法**：Sonnet 5 面向 browser、planning、coding 与 knowledge work，并下放到免费及企业用户。
- **效果**：Anthropic 称其接近 Opus 4.8；社区对成本、latency 和 regression 反馈分化，尚无稳定独立横评。[Axios](https://www.axios.com/2026/06/30/anthropic-sonnet-5-agents-mythos-fable)

### 2. Fable 5 的恢复访问
- **背景**：高 cyber 能力模型此前受政府审核限制。
- **问题**：供应中断会破坏绑定单一 provider 的生产工作流。
- **方法**：恢复多产品入口，并继续使用更强 safeguards 与访问控制。
- **效果**：服务可用性恢复，但长期审核规则仍不透明，企业仍需 fallback。[来源](https://www.kubiosec.tech/blog/2026-07-03-AgenticUpdates)

### 3. Gemini 3.5 Pro 的质量 refinement
- **背景**：frontier labs 在快速发布与企业可靠性之间承压。
- **问题**：早期测试暴露的稳定性问题可能被 benchmark headline 掩盖。
- **方法**：据报道 Google 延后 GA 以继续质量调整。
- **效果**：暂无正式 model card 和最终 GA 数据，不能判断改进幅度。[来源](https://www.kubiosec.tech/blog/2026-07-03-AgenticUpdates)

### 4. MCP stateless core
- **背景**：MCP server 从本地连接器进入云端多租户环境。
- **问题**：session-heavy 架构不利于扩缩容、恢复和权限治理。
- **方法**：RC 将无状态核心与 Tasks、Apps 等扩展分层。
- **效果**：架构更适合生产部署，但迁移和互操作仍待验证。[MCP](https://github.com/modelcontextprotocol/modelcontextprotocol/releases/tag/2026-07-28-RC)
